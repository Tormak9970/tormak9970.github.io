Set Prefix

Description

Used to set the bot's prefix. This is used as a, well, prefix for all the commands

Command Help

Description

Used to provide help for questions about any commands.
$help lists all commands.
$help cmd provides more details on the specified command

Bot Info

Description

This command provides information on the bot such as the purpose, creator and date created among other information.

Server Info

Description

This command provides information on the server such as the owner, date created, region and number of members among other information.

User Info

Description

This command provides information on the specified user such as the date joined, roles and nickname among other information.

Ban Word

Description

This command is used to ban words that you don't want to allow people to use. Commonly this would be curse words, age, real name etc.

Get Banned Words

Description

This command is used to view the list of banned words for the server.

Remove Banned Word

Description

This command is used to unban a word. By default, this command can only be used by admins and users with the MANAGE_SERVER permission

Ban User

Description

This command is used to ban a user, and it requires you to confirm that you wish to ban them. By default, this command can only be used by admins and users with the BAN_MEMBERS permission

Kick User

Description

This command is used to kick a user. By default, this command can only be used by admins and users with the KICK_MEMBERS permission

Mute User

Description

This command is used to mute a user for the specified amount of time.

Reaction Roles

Description

This command is used to create a reaction role. By default, this command can only be used by admins and users with the MANAGE_SERVER permission

OnJoin/Auto Roles

Description

This command is used to create an OnJoin/Auto role. By default, this command can only be used by admins and users with the MANAGE_SERVER permission

Remove OnJoin/Auto Role

Description

This command is used to remove an OnJoin/Auto role. By default, this command can only be used by admins and users with the MANAGE_SERVER permission

User Variables

{user}

Usage

When used, {user} is replaced with the sender's effective name. The user's effective name means their nickname(if they have one), otherwise it is their username.

Custom Command:

Welcome {user}!


Bot's Response:

Welcome Modbot!


{user.id}

Usage

When used, {user.id} is replaced with the sender's id. This isnt really useful to the average person, but you could do something fun like set everyone's nicknames to their id.

Custom Command:

Your id is {user.id}


Bot's Response:

Your id is 1518910274738385


{user.name}

Usage

When used, {user.name} is replaced with the sender's name. This is their full discord name, so username plus the discriminator(hastag and numbers after it).

Custom Command:

Your name is {user.name}


Bot's Response:

Your name is Wumpus#9538


{user.username}

Usage

When used, {user.username} is replaced with the sender's username. This is their full discord name, without the discriminator(hastag and numbers after it).

Custom Command:

Your username is {user.username}


Bot's Response:

Your username is Wumpus


{user.discriminator}

Usage

When used, {user.discriminator} is replaced with the sender's discriminator(hastag and numbers after it in your name).

Custom Command:

Your discriminator is {user.discriminator}


Bot's Response:

Your discriminator is #9538


{user.nick}

Usage

When used, {user.nick} is replaced with the sender's nickname in this server, if applicable, or their username.

Custom Command:

Your name in this server is {user.nick}


Bot's Response:

Your name in this server is The Discord Mascot


{user.game}

Usage

When used, {user.game} is replaced with the sender's current game activity.

Custom Command:

You are currently playing: {user.game}


Bot's Response:

You are currently playing: Rocket League


{user.avatar}

Usage

When used, {user.avatar} is replaced with a url of sender's avatar.

Custom Command:

Your avatar is: {user.avatar}


Bot's Response:

Your avatar is: https://media.tenor.com/images/fa7ac0f660787f24e8a925dae120be7a/tenor.png


{user.mention}

Usage

When used, {user.mention} is replaced with a mention of the sender.

Custom Command:

Welcome {user.mention}


Bot's Response:

Welcome @Wumpus


{user.createdAt}

Usage

When used, {user.createdAt} is replaced with the date the user created their account.

Custom Command:

You created your account on: {user.createdAt}


Bot's Response:

You created your account on: May 13, 2015


{user.joinedAt}

Usage

When used, {user.joinedAt} is replaced with the date the user joined this server.

Custom Command:

You joined this server on: {user.joinedAt}


Bot's Response:

You joined this server on: January 20, 2018

Miscellaneous Variables

{role} (put role name here)

Usage

When {role} (put role name here) is used, both words are replaced with a mention of the role(the role must have Anyone Can Mention enabled).

Custom Command:

{role} youtube, there is a new youtube video out!


Bot's Response:

@Youtube, there is a new youtube video out!


{everyone}

Usage

When used, {everyone} it is replaced with @everyone(pings everyone in the server).

Custom Command:

{everyone} happy holidays


Bot's Response:

@everyone happy holidays


{here}

Usage

When used, {here} it is replaced with @here(pings everyone who is online).

Custom Command:

{here} happy holidays


Bot's Response:

@here happy holidays



Advanced Variables

{prefix}

Usage

When used, {prefix} is replaced with the bot's command prefix.

Custom Command:

The bot's current prefix is {prefix}


Bot's Response:

The bot's current prefix is $


{delete}

Usage

When used, {delete} is removed and the discord message that triggered the command is deleted.

Custom Command:

{delete}I have deleted your command msg


Bot's Response:

I have deleted your command msg(the discord message you sent to trigger the response would be gone).


{silent}

Usage

When used, {silent} is removed but bot does not respond.

Custom Command:

{silent}, there is a new youtube video out!


Bot's Response:


$N

Usage

When used, $N is replaced with the first parameter you pass(it will throw an error if you do not pass enough parameters).

Custom Command:

The parameter you provided is $N


Bot's Response:

The parameter you provided is testingThisCommand(the msg you would send to get this respons would be: !commandName testingThisCommand)


{require-role}: (role name)

Usage

When {require-role}: (role name) is used, both words are removed and the bot will only respond if the command is sent by a user with the specified role.(This can not be used with a role also used in a {not-role}:)

Custom Command:

{require-role}: AdminThis is an Admin test command


Bot's Response:

This is an Admin test command(message would only appear if the command was sent by someone with the Admin role)


{require-channel}: (channel name)

Usage

When {require-channel}: (channel name) is used, both words are removed and the bot will only respond if the command is sent in the specified channel.(This can only be used 1 time per command. Also, this can not be used with a channel also used in a {not-channel}:)

Custom Command:

{require-channel}: bot-commandsThis is a test command


Bot's Response:

This is a test command(message would only appear if the command is sent in the bot-commands channel)


{not-role}: (role name)

Usage

When {not-role}: (role name) is used, both words are removed and the bot will not respond if the command is sent by a user with the the specified role.(this can not be used with a role also used in a {require-role}:)

Custom Command:

{not-role}: unverifiedThis is a test command


Bot's Response:

This is a test command(message would not appear if the command is sent by a user with the unverified role)


{not-channel}: (channel name)

Usage

When {not-channel}: (channel name) is used, both words are removed and the bot will not respond if the command is sent in the specified channel.(this can not be used with a channel also used in a {require-channel}:)

Custom Command:

{not-channel}: bot-commandsThis is a test command


Bot's Response:

This is a test command(message would not appear if the command is sent in the bot-commands channel)


{dm}

Usage

When used, {dm} is removed and the bot's response is direct messaged to the sender.(this can not be used with {respond}:)

Custom Command:

{dm}What command would you like help with?


Bot's Response:

What command would you like help with?(message would appear in the sender's direct messages)


{respond}: (channel name)

Usage

When {respond}: (channel name) is used, both words are removed and the bot's response is sent in the specified channel.(this can not be used with {dm})

Custom Command:

{respond}: bot-commandsThis is a test command


Bot's Response:

This is a test command(message would appear in the bot-commands channel)


{choose}: (word/number) (word/number) (word/number)

Usage

When used, {choose}: (word/number) (word/number) (word/number) will randomly choose one of the 3 items provided(note that, 1, you must provide 3 items, and 2, that unless you have a {choice} later in the response, it will not do anything).

Custom Command:

{choose}: 1 2 3 I chose {choice}


Bot's Response:

I chose 3


{choice}

Usage

When used, {choice} is replaced with the choice from the 3 provided( it is random, and requires a {choose}: to have been used before it).

Custom Command:

{choose}: 1 2 3 I chose {choice}


Bot's Response:

I chose 2
