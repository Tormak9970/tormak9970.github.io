About

The SWTOR Slicers Collaboration is a group who enjoys taking Star Wars The Old Republic beyond the game. Whether this be through using models for fan films, or creating tools for the community (such as my Character and Outfit Designer), the community enjoys finding new and unique was to utilize the game.

SWTOR Slicers

SWTOR Character and Outfit Designer

Overview

This is one of my favorite projects. It is an external character creator and outfit designer for Star Wars The Old Republic. I really love this game and the community, and wanted to do something everyone could use and enjoy. It is very complexed, and I had to learn alot of new topics and skills while working on it. Im also very proud of it since it is the first tool of its kind made by anyone, ever, and that required me to solve alot of problems no one else has done before.

Reasons & Purpose

  • I wanted to make something truly unique that had not been done before
    • This is the only Star Wars The Old Republic external character designer
  • I also wanted to make something people would not only want to use, but also something that would inspire people, and help them realise what they can do.
  • I had read comments and posts on a variety of platforms from people requesting/asking about a character viewer/creator, the earliest dating back to around 2014
  • I also wanted to expedite the process from re-creating your character outside of the game (for 3D modeling, printing, or just for fun).
    • The process could take upwards of one to two hour for one character
    • With my program, it takes ~20 minutes

Outcome

  • How to read hexadecimal headers
  • How to read custom file formats
  • Basics of PHP
  • Advanced JavaScript
  • How to better manage the layout of my code
  • How to better structure and organize my time
  • How to break down problems into achievable sections
  • How to effectively troubleshoot issues and debug
  • Sometimes the best solution is to take a break and work on different part/something else for a while
  • How to use the THREE.js library
  • Basics of 3D design/modeling
    • HLSL
    • GLSL
    • The graphics pipeline (how 3D models are shaded and rendered)
    • How models work
    • How textures work
    • How lighting works
    • How shadows work

Release two (March 25, 2021)

Length: ~300 hours

Bug Fixes

  • Fixed many broken items (only a few left still needing to be fixed)
  • Fixed a few edge cases where things like hoods would render on Togrutas and Twi'leks
  • Fixed an issue where exporting the zip had no file extension on Firefox
  • Fixed long loading times

New Features

  • Now has dark side corruption
  • Added Eye Cube Maps (the glossy reflection on the character's eyes)
  • Changed the lighting to be closer to the preview window. (this is the default for now, but more on that later)
  • Added a Unify Colors option for each armor slot (this makes the slot in question become the same color as the chest slot)
  • Added ability to apply dyes for each slot
  • Added tick marks to make it easier to tell where to click to move the slider to a specific value
  • Re-arranged the order of a few customizations to match in-game numbering
  • Updated many of the UI images to make them clearer
  • Now have a randomize button, and it also randomizes the slider values on load.
  • Tweaked a bunch of the rendering settings (camera position, lighting angle, lighting intensity, etc.)

Release one (March 14, 2021)

Length: 270+ hours

New Features

  • Users can download a .zip with all the nessesary info to re-create their character in blender
  • Has all character creator options that the game has
    • Customization sliders
    • Gender
    • Classes
    • Species
  • Allows changing of light-color gradient
  • Users can search by keywords to get a list of every related item, which they can then select from to display one
  • Users can add a global dye to their outfit
  • Users can share their creations using the sharing url button at the bottom

Modbot

Overview

Modbot was a project I started after I had finished learning Java and was familiar with JavaScript and SpringBoot. This allowed me to make a more advanced project, as I already knew the foundations. For this project, I decided to split the Database from the Discord Bot. This meant that I needed to learn how to send Java to Java JSON data. I also made a frontend for users to interact with, which was my second web developement project.

Reasons & Purpose

  • I wanted to make a very versatile and complexed discord bot, with oAuth, custom commands, and able to be tailored to a user's preferences
  • I wanted to practice more with SpringBoot, and learn some of its more advanced features.
  • I wanted to practice web developement.

Outcome

  • Learned the more advanced parts of SpringBoot
  • Learned how to structure HTTP requests in java, and decode JSONs in java using GSON
  • Was able to make a very advanced discord bot, that works reliably.
  • Learned the basics of oAuth and good conventions to use when implimenting it.

Components

Modbot is composed of three(3) main parts.

API

The API is the core of the project. This is where all of the custom command information is stored, as well as the settings for which commands are enabled/disabled.

Front End

The frontend is how a user can interact with the bot, and change settings and features.

Discord Bot

The Modbot discord bot is designed to help a person moderate their discord communities. It allows users to interact with it using the web UI.
